# 任务管理与交接模板

本文件管理 AI 辅助开发会话的任务连续性、会话过渡和知识转移。

## 用途

此模板有助于维护:
- AI 开发会话之间的**会话连续性**
- 复杂的、多会话工作的**任务状态跟踪**
- 团队成员切换时的**上下文保留**
- 项目交接的**知识转移**
- 持续开发工作的**进度文档**

## 当前会话状态

### 活动任务
记录当前正在进行的工作:

```markdown
## 进行中
- [ ] 任务 1:[简要描述]
  - 状态:[已开始/已阻塞/等待审查]
  - 上下文:[相关文件、已做决策]
  - 下一步:[接下来需要做什么]
  - 依赖:[此任务依赖于什么]

- [ ] 任务 2:[简要描述]
  - 状态:[当前状态]
  - 修改的文件:[已更改文件列表]
  - 挑战:[遇到的任何问题]
  - 备注:[继续所需的重要上下文]
```

### 待办任务
记录排队的工作:

```markdown
## 待办
- [ ] 任务 A:[描述]
  - 优先级:[高/中/低]
  - 依赖:[必须先完成什么]
  - 预计工作量:[时间估计]
  - 上下文:[背景信息]

- [ ] 任务 B:[描述]
  - 优先级:[优先级别]
  - 需求:[具体要求或约束]
  - 所需资源:[所需的工具、访问权限、信息]
```

### 已完成任务
跟踪已完成的工作以保留上下文:

```markdown
## 本会话已完成
- [x] 任务 X:[描述]
  - 完成:[日期]
  - 成果:[完成了什么]
  - 修改的文件:[修改的文件]
  - 备注:[重要决策或经验教训]

- [x] 任务 Y:[描述]
  - 完成:[日期]
  - 影响:[这如何影响其他任务]
  - 需要跟进:[需要的任何后续行动]
```

## 架构与设计决策

### 最近的决策
记录开发过程中做出的架构决策:

```markdown
## 已做的设计决策
- **决策**:[决定了什么]
  - 日期:[何时做出决策]
  - 理由:[为什么选择这种方法]
  - 考虑的替代方案:[评估的其他选项]
  - 影响:[这如何影响系统]
  - 验证:[如何验证这是正确的选择]

- **决策**:[另一个决策]
  - 上下文:[导致此决策的情况]
  - 权衡:[这个选择获得/失去了什么]
  - 依赖:[此决策依赖于什么]
```

### 技术债务与问题
跟踪技术债务和已知问题:

```markdown
## 已识别的技术债务
- **问题**:[技术债务描述]
  - 位置:[代码库中的位置]
  - 影响:[如何影响开发/性能]
  - 建议的解决方案:[如何解决它]
  - 优先级:[何时应该解决]

- **问题**:[另一个问题]
  - 根本原因:[为什么存在这种债务]
  - 变通方法:[当前的缓解策略]
  - 长期修复:[适当的解决方案方法]
```

## 下一会话目标

### 即时优先事项
定义接下来应该处理什么:

```markdown
## 下一会话优先事项
1. **主要目标**:[下一会话的主要目标]
   - 成功标准:[如何知道这已完成]
   - 先决条件:[事先必须准备什么]
   - 预计工作量:[时间估计]

2. **次要目标**:[次要目标]
   - 依赖:[这依赖于什么]
   - 所需资源:[所需的工具、信息、访问权限]

3. **时间允许的话**:[可选任务]
   - 上下文:[为什么这些有价值的背景]
   - 准备:[开始这些需要做什么]
```

### 知识差距
记录需要研究或澄清的领域:

```markdown
## 需要解决的知识差距
- **问题**:[需要澄清什么]
  - 影响:[这如何影响当前工作]
  - 所需研究:[需要什么调查]
  - 决策者:[谁可以回答这个]

- **未知**:[技术不确定性]
  - 选项:[要探索的可能方法]
  - 实验:[应该测试什么]
  - 时间线:[何时需要解决]
```

## 继续的上下文

### 关键文件与组件
记录会话连续性的重要文件:

```markdown
## 当前正在修改的文件
- `[文件路径]`:[目的和当前更改]
- `[文件路径]`:[这里正在实现什么]
- `[文件路径]`:[状态和下一步]

## 重要的上下文文件
- `[上下文文件]`:[为什么这相关]
- `[文档]`:[这包含什么信息]
- `[参考]`:[这如何与当前工作相关]
```

### 开发环境
记录环境和设置考虑事项:

```markdown
## 环境状态
- **开发设置**:[开发环境的当前状态]
- **数据库**:[模式更改、迁移、数据状态]
- **外部服务**:[API 密钥、服务配置]
- **测试**:[测试套件状态、覆盖率、失败的测试]
- **构建/部署**:[构建状态、部署考虑事项]
```


---

*此模板为管理任务连续性和知识转移提供了全面的框架。根据你团队的工作流程、项目复杂性和沟通需求进行自定义。*
